CRITICAL DESIGN PRINCIPLE: WORD FATIGUE AVOIDANCE
Rule: On crisp, minimal slides (title slides, section dividers, any slide with <50 words), NEVER repeat words across different text elements.

Why: Word repetition creates fatigue, reduces impact, makes slides feel lazy.

Examples of violations to AVOID:

âŒ Tagline: "Control Your Data" â†’ Pillar: "Take control of your data at the source"
âŒ Headline: "Speed your team" â†’ Bullet: "Speed your deployment"
âŒ Pillar 1: "Reduce costs" â†’ Pillar 2: "Reduce complexity"
How to avoid:

Use synonyms: Control â†’ Command â†’ Manage â†’ Own
Vary sentence structure: "Control costs" vs "Reduce expenses" vs "Cut spending"
Use different parts of speech: "Control" (verb) vs "Governance" (noun)
Check for repetition before finalizing
Prioritize most important usage first
Exceptions (acceptable repetition):

Company/product names (Expanso, Snowflake, etc.)
Technical terms with no clear synonym (Kubernetes, API)
Intentional rhetorical repetition for emphasis ("Data. Data. Data.")
Applies to:

Title slides
Section divider slides
Any slide with <50 words total
Slides where every word is highly visible (large fonts, minimal text)
THE 10/20/30 RULE (ALWAYS ENFORCE)
Guy Kawasaki's presentation rule - NON-NEGOTIABLE:

10 SLIDES MAXIMUM - No presentation should exceed 10 slides (excluding appendix)
20 MINUTES MAXIMUM - Content deliverable in 20 minutes, leaving time for discussion
30 POINT FONT MINIMUM - All body text must be 30pt or larger (headlines 60-96pt)
Implications:

Ruthlessly minimize text - if it doesn't fit in 30pt font, it doesn't belong on slide
One idea per slide - force clarity
Visual over verbal - use diagrams instead of text blocks
No bullet point essays - maximum 3-5 short bullets per slide
Speaker notes, not slide text - detailed info goes in notes
Challenge every line: "Does audience NEED to read this, or can speaker say it?"
Exceptions (rare):

Appendix/reference slides beyond the 10 (clearly labeled)
Legal disclaimers (minimize, use smallest acceptable size)
Technical specifications (use tables/diagrams, not paragraphs)

SLIDE STRUCTURE RULES
General Slide Anatomy
Every slide MUST have:

One clear headline (what is this slide about?)
One focal point (what should audience look at first?)
Minimal supporting text (3-5 bullets maximum, or 1-2 short sentences)
Optional visual element (diagram, chart, icon - only if adds clarity)
Consistent footer (optional: slide number, presenter name)
Every slide MUST NOT have:

Multiple competing focal points
More than one concept or message
Paragraphs of text
Decorative elements that don't serve the message
Cluttered layouts with <40% whitespace
Text smaller than 30pt (body) or 19pt (footer)

TROUBLESHOOTING COMMON ISSUES
"My slide feels cluttered"
Solutions:

Remove 50% of the text - if you can say it, don't write it
Increase whitespace - add more margin around elements
Split into 2 slides - one concept per slide rule
Remove decorative elements - every element must serve the message
Increase font size - forces you to use fewer words
"My fonts look too small"
Solutions:

Check actual point size - must be 30pt minimum for body text
Reduce amount of text - don't try to fit paragraphs
Use hierarchy - not everything needs to be same size
Test from distance - stand 20 feet away, can you read it?
Remove the content - if it doesn't fit in 30pt, it doesn't belong
"I can't fit my message in the space"
Solutions:

Simplify the message - what's the ONE thing to remember?
Use speaker notes - detailed explanation goes in notes, not on slide
Split into multiple slides - better 2 clear slides than 1 cluttered
Use a diagram - visual can replace 100 words
Question if you need it - does this slide serve the narrative?
"The colors don't look right"
Solutions:

Check hex values - must match brand guidelines exactly
Check color profile - should be sRGB for screen
Check monitor calibration - colors may look different on different screens
Use color picker - sample from reference slides to ensure match
Export and test - view on multiple devices to verify
"The gradient has banding"
Solutions:

Increase bit depth - use 16-bit color if possible
Add noise texture - 3% opacity hides banding
Use more gradient stops - add intermediate color at 25% and 75%
Export at higher resolution - 2x or 3x native resolution
Use PNG instead of JPEG - lossless compression preserves gradients
"Fonts are rendering differently"
Solutions:

Embed fonts - ensure Inter is embedded in file
Check font installation - verify Inter is installed on system
Use web fonts - for web-based presentations, load from Google Fonts 4

WORKFLOW & PROCESS
Step 1: Planning (Before Creating Slides)
Questions to answer:

Who is the audience? (Persona: CFO, Engineering, CDO, Multi-persona)
What is the goal? (Awareness, Consideration, Decision)
How much time? (5 min, 10 min, 20 min, 45 min)
What format? (Live, PDF, Video, Conference)
What's the one thing they should remember?
Outline the narrative:

What problem are we solving?
Why does it exist?
What's our solution?
How does it work?
What proof do we have?
What's the next step?
Create slide titles first:

Write headline for each slide
Ensure logical flow from one to next
Verify you're under 10 slides
Check that each slide has ONE concept
Step 2: Content Creation
For each slide:

Write the headline (60-72pt)
Identify the focal point (what should they look at?)
Add supporting text (30-36pt, 3-5 bullets max)
Consider if visual would help (diagram, chart, icon)
Write speaker notes (what will you say?)
Content checklist per slide:

 One concept only
 Headline is clear and specific
 Text is minimal (can read in 3 seconds)
 No word repetition on this slide
 Active verbs used
 Metrics are specific (not vague)
 Supports overall narrative
Step 3: Visual Design
For each slide:

Apply navy-to-charcoal gradient background
Set headline (left-aligned for most, centered for title slide)
Set body text (left-aligned, 30pt minimum)
Add visual element if needed (diagram, metric, icon)
Add footer if needed (presenter info, slide number)
Check whitespace (50-65%)
Check focal point (one element dominates)
Design checklist per slide:

 Background gradient applied
 Fonts are Inter family
 All body text 30pt or larger
 Colors match brand palette
 One focal point
 50-65% whitespace
 Readable from 20 feet
Step 4: Review & Refinement
Self-review:

Read through entire deck without stopping
Time yourself - under 20 minutes?
Stand back 20 feet - can you read everything?
Check for word repetition across slides
Verify all metrics are cited correctly
Run quality checklist (see earlier section)
Peer review:

Show to colleague who doesn't know the content
Ask: "What's the main message?"
Ask: "What's confusing?"
Ask: "What would you remove?"
Incorporate feedback selectively
Final polish:

Fix any typos or grammatical errors
Ensure consistent spacing/alignment
Verify all links work (if interactive)
Check export quality on target device
Save final version with clear filename
Step 5: Delivery Preparation
Practice:

Rehearse full presentation 3 times minimum
Time yourself each time - adjust pacing
Practice transitions between slides
Prepare for anticipated questions
Record yourself - watch for filler words, pacing issues
Technical setup:

Test on actual presentation equipment
Have backup (PDF, USB drive, cloud link)
Test screen mirroring/projection
Verify fonts render correctly
Check that animations work (if using)
Day of presentation:

Arrive early to test setup
Have water available (dry mouth during presenting)
Silence phone/notifications
Open presenter notes on second screen
Take deep breath, start confidently

FINAL NOTE
This framework is comprehensive, but don't let perfection block progress.

The most important rules:

10/20/30 rule - 10 slides, 20 minutes, 30pt fonts
One concept per slide - clarity over completeness
Specific metrics - "58% reduction" not "significant savings"
No word repetition - avoid fatigue on minimal slides
Whitespace - 50-65% empty space signals confidence
Start with these five rules. Everything else is refinement.

When in doubt, ask: "Does this serve the message, or am I just filling space?"
