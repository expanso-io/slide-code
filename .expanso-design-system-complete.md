## Brand Identity

### Core Principles

- **Modern & Clean**: Avoid visual clutter; use whitespace strategically
- **Accessible**: Follow WCAG 2.1 AA standards minimum
- **Adaptable**: Design system works across digital and print
- **Purposeful**: Every design choice serves the user experience

### Brand Positioning

**Expanso (Violet)**: Innovation, creativity, forward-thinking. Violet embodies the balance between stability (blue) and energy (red), representing thoughtful precision and bold action.

**Bacalhau (Blue/Ice)**: Trust, reliability, vast potential. The ocean metaphor (bacalhau = cod in Portuguese) represents depth, exploration, and boundless possibilities in distributed computing.

### Brand Rules (Critical)

1. ✅ **Violet MUST appear** in all Expanso designs
2. ✅ **Blue/Ice MUST appear** in all Bacalhau designs  
3. ✅ When combining palettes, one must be **dominant** (>50%)
4. ✅ All three (Violet, Blue, Ice) **can coexist** in any design
5. ❌ **Never use color alone** to convey information
6. ✅ Always **test contrast ratios** (min 4.5:1 for text)

## Typography

### Type Scale (Modular 1.25 ratio)

```
Display:    48px / 3rem      weight: 700    line-height: 1.1
H1:         38px / 2.375rem  weight: 700    line-height: 1.2
H2:         30px / 1.875rem  weight: 600    line-height: 1.3
H3:         24px / 1.5rem    weight: 600    line-height: 1.4
H4:         19px / 1.1875rem weight: 600    line-height: 1.4
Body:       16px / 1rem      weight: 400    line-height: 1.6
Small:      14px / 0.875rem  weight: 400    line-height: 1.5
Caption:    12px / 0.75rem   weight: 400    line-height: 1.4
```

### Presentation Typography (Scaled Up)

For presentations, scale significantly for readability:

```
Display:    96px / 6rem      weight: 700    line-height: 1.1
H1:         72px / 4.5rem    weight: 700    line-height: 1.2
H2:         60px / 3.75rem   weight: 600    line-height: 1.3
H3:         48px / 3rem      weight: 600    line-height: 1.4
Body:       32px / 2rem      weight: 400    line-height: 1.6
Small:      28px / 1.75rem   weight: 400    line-height: 1.5
Caption:    22px / 1.375rem  weight: 400    line-height: 1.4
```

**Note**: Follow 10/20/30 rule - minimum 30pt for body text in presentations.

### Font Families

**System Font Stack** (recommended for UI):
```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto,
             "Helvetica Neue", Arial, sans-serif;
```

**Alternative Stacks**:
- **Modern Sans**: Inter, "SF Pro", system-ui, sans-serif
- **Monospace**: "SF Mono", Monaco, "Cascadia Code", Consolas, monospace

### Font Weights
```
Light: 300 (use sparingly, accessibility concerns)
Regular: 400 (body text)
Medium: 500 (emphasis, labels)
Semibold: 600 (subheadings, buttons)
Bold: 700 (headings, strong emphasis)
```

### Typography Guidelines

- **Line length**: 45-75 characters for optimal readability
- **Paragraph spacing**: 1.5em between paragraphs
- **Letter spacing**: Default for body, -0.02em for headings
- **Heading hierarchy**: Never skip levels (H1 → H2 → H3)
- **Emphasis**: Use weight changes over italics for UI elements

8px / 0.5rem → sm (compact spacing)
12px / 0.75rem → md (default spacing)
16px / 1rem → base (comfortable spacing)
24px / 1.5rem → lg (section spacing)
32px / 2rem → xl (major sections)
48px / 3rem → 2xl (page sections)

#### Grid System
- **Desktop**: 12-column grid, 1200px max-width
- **Tablet**: 8-column grid, 768px max-width

- **Margin**: 32px (desktop), 16px (mobile)

## Accessibility

### Contrast Requirements (WCAG 2.1)

#### Level AA (Minimum)
- **Normal text** (< 18px): 4.5:1 minimum
- **Large text** (≥ 18px or ≥ 14px bold): 3:1 minimum
- **UI components**: 3:1 minimum

#### Level AAA (Recommended)
- **Normal text**: 7:1 minimum
- **Large text**: 4.5:1 minimum

### Pre-validated Combinations

**On White (#FFFFFF) or grey-50 (#FCFCFC)**:
```
✓ grey-900 (#000000): 21:1
✓ grey-800 (#454545): 8.59:1
✓ grey-700 (#727272): 5.74:1
✓ violet-900 (#27065B): 15.2:1
✓ violet-800 (#501E99): 8.1:1
✓ violet-700 (#6823CD): 6.2:1
✓ blue-900 (#000F3B): 18.5:1
✓ blue-800 (#001C70): 12.3:1
```

**On Black (#000000) or grey-900**:
```
✓ #FFFFFF: 21:1
✓ grey-50 (#FCFCFC): 19.8:1
✓ violet-200 (#DFCBFD): 12.1:1
✓ violet-100 (#F0E6FF): 16.5:1
✓ blue-200 (#BED3FF): 14.2:1
✓ ice-300 (#95E5FF): 13.5:1
```

### Accessible Color Usage

#### Don't Rely on Color Alone
- Add icons to status messages
- Use patterns in charts/graphs
- Include text labels with colored indicators

## Illustration Style

### Art Style & Aesthetic

**Visual Approach**: Illustrative, hand-drawn cartoon/doodle aesthetic with slightly whimsical yet detailed character.

**Key Characteristics**:
- Strong, clear, dark outlines for definition
- Limited, earthy color palette (muted tones)
- Soft blues, greys, browns, greens, creams/beiges
- Accents of soft orange, pale yellow, light red (sparingly)
- **Avoid**: Bright, neon, or overly vibrant colors

### Background & Consistency

- **Backgrounds**: Light, slightly textured, desaturated tones
- **Texture**: Subtle cross-hatching, pastel shading for depth
- **Watercolor-like**: Light washes for atmosphere

### Character Design

- **Robots**: Simple, expressive with visible joints/pipes
- **Expressions**: Worried, contemplative, determined
- **Animals/Humans**: Small industrious creatures or simplified figures

### Overall Mood

Slightly humorous or lighthearted, conveying serious/complex themes accessibly. Gentle chaos or organized complexity.

### What to Avoid
- Photorealism or overly detailed textures
- Complex human faces or realistic anatomy
- Aggressive or overly dark/ominous tones
- Text within images (unless specifically requested)
- Overly complex geometric patterns

## Best Practices

### Do's

✓ Use violet in all Expanso designs  
✓ Use blue/ice in all Bacalhau designs  
✓ Maintain one dominant color palette (>50%)  
✓ Test contrast ratios before finalizing  
✓ Use semantic colors consistently  
✓ Provide adequate white space  
✓ Follow the spacing scale (multiples of 4px)  
✓ Include focus indicators for all interactive elements  
✓ Design mobile-first, then scale up  

### Don'ts

✗ Don't use color as the only differentiator  
✗ Don't skip heading levels  
✗ Don't use light text (< 400 weight) for small sizes  
✗ Don't overcrowd layouts with too many colors  
✗ Don't ignore accessibility standards  
✗ Don't create custom colors outside the palette  
✗ Don't use more than 3-4 accent colors simultaneously
